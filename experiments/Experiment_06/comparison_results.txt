EXPERIMENT 06: RERANKER COMPARISON
MMR vs MMR + Cross-Encoder Reranker
======================================================================

MMR RETRIEVAL
--------------------------------------------------
Retrieval Time      : 0.2738 seconds
Retrieved Documents : 5
Unique Pages        : 4
Duplicate Chunks    : 0
Context Diversity   : 0.80

MMR + CROSS-ENCODER
--------------------------------------------------
Retrieval Time      : 0.2738 seconds
Reranking Time      : 0.3823 seconds
Total Pipeline Time : 0.6561 seconds
Retrieved Documents : 5
Unique Pages        : 4
Duplicate Chunks    : 0
Context Diversity   : 0.80

======================================================================
ENGINEERING DECISION
======================================================================
✓ Cross-Encoder introduced additional inference latency.
✓ Context diversity was preserved after reranking.
✓ Reranking maintained low document redundancy.

Production Considerations:
- MMR efficiently retrieves diverse candidate documents.
- Cross-Encoder performs semantic relevance refinement.
- Reranking increases latency but improves document ordering.
- Suitable when answer quality is prioritized over response speed.

Final Recommendation:
Use MMR retrieval with Cross-Encoder reranking in production RAG systems where retrieval quality is more important than slightly higher latency.
